# METHOD.md

## 1. Agent 方法论

升级后的 Agent 不再是单点文案生成，而是私域运营闭环：

```text
飞书营销表同步
-> 产品信息结构化
-> 运营反馈学习
-> 自动找配图
-> 文案生成与去 AI 味
-> 自动排版
-> 发布前检查
-> 推荐发布版本
```

## 2. 节点设计

| 节点 | 接口 | 输入 | 输出 |
|---|---|---|---|
| 飞书同步 | `/api/feishu/sync` | appToken, tableId, viewId | 营销表、素材表、品类表现 |
| 信息整理 | `/api/product/normalize` | 产品信息、活动信息 | 结构化产品卡 |
| 自动找配图 | `/api/assets/search` | 产品类型、关键词、素材来源 | 候选素材、匹配分、首图 |
| 文案生成 | `/api/copy/generate` | 产品卡、反馈数据、禁用词 | 3 条文案 |
| 自动排版 | `/api/poster/layout` | 首图、推荐文案、样式 | 排版预览 |
| 反馈学习 | `/api/ops/learn` | 曝光、评论、私聊、成交、沟通摘要 | 调优建议 |
| 发布检查 | `/api/publish/check` | 文案、素材、活动、禁用词 | 风险检查和推荐 |

## 3. 飞书连接方法

真实版本可以用两种方式：

1. **飞书开放平台 API**：通过多维表格 records/search 读取营销表、素材表。
2. **飞书自动化 Webhook**：表格更新后推送到 Agent 后端。

当前本地原型保留连接字段，用模拟数据展示完整链路。

## 4. 自动配图方法

配图不是给建议，而是 Agent 执行检索：

1. 读取产品类型、卖点、关键词、潜客反馈。
2. 从飞书素材表或本地素材库中取候选素材。
3. 根据类型匹配、关键词匹配、客户反馈匹配打分。
4. 选出首图和补充图。
5. 交给排版节点生成海报预览。

## 5. 运营反馈学习方法

输入：

- 曝光
- 点赞
- 评论
- 私聊
- 成交
- 热卖品类
- 潜在客户沟通摘要

计算：

- 互动率 = (点赞 + 评论 + 私聊) / 曝光
- 私聊率 = 私聊 / 曝光
- 成交率 = 成交 / 曝光

输出：

- 文案开头是否需要加强场景
- 结尾是否需要把评论导向私聊
- 是否补充活动截止、库存、试用方式
- 首图是否应选择实拍、细节图或场景图

## 6. 当前原型实现

当前小应用用 `app.js` 实现本地模拟版：

- `simulateFeishuSync()`：模拟飞书同步
- `learnFromOps()`：运营反馈学习
- `searchAssets()`：自动找配图
- `buildLayout()`：自动排版
- `generateCopyDrafts()`：生成文案
- `checkPublication()`：发布检查

这些函数都可以在真实项目中替换为后端 API 或大模型节点。
