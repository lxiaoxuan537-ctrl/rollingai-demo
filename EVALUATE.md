# EVALUATE.md

## 1. 评估目标

验证这个 AI Agent 是否能完成真实私域运营闭环：

```text
飞书数据 -> 自动找图 -> 文案生成 -> 自动排版 -> 发布检查 -> 反馈学习
```

## 2. 功能验收

| 验收项 | 标准 | 当前状态 |
|---|---|---|
| 飞书连接配置 | 支持输入 appToken、tableId、viewId | 已完成 |
| 飞书同步模拟 | 能返回营销表、素材表、品类数据 | 已完成 |
| 表单输入 | 支持产品、卖点、人群、活动等 | 已完成 |
| 卖点提炼 | 能输出适合人群、核心卖点、切入角度 | 已完成 |
| 自动找配图 | 能根据类型和关键词选素材 | 已完成 |
| 自动排版 | 能生成朋友圈海报预览 | 已完成 |
| 多风格文案 | 能生成 3 条不同风格朋友圈文案 | 已完成 |
| 去 AI 味 | 能规避模板词和过度营销表达 | 已完成 |
| 运营调优 | 能读取曝光、评论、私聊、成交并给建议 | 已完成 |
| 发布检查 | 能提示风险词、长度、素材、活动完整性 | 已完成 |
| 接口链路 | 左侧节点可展示接口契约 | 已完成 |

## 3. 内容质量评估

| 指标 | 说明 |
|---|---|
| 真人感 | 是否像运营或玩家发朋友圈 |
| 场景感 | 是否出现开黑、排位、桌搭、宿舍等具体场景 |
| 配图匹配 | 首图是否匹配产品类型和用户反馈 |
| 排版可用 | 是否能作为朋友圈首图或海报方向 |
| 调优有效 | 是否根据评论、私聊、成交提出下一步建议 |
| 风险控制 | 是否避免绝对化、虚假承诺和活动信息缺失 |

## 4. 测试用例

| 用例 | 产品 | 重点验证 |
|---|---|---|
| Case 1 | 无线电竞鼠标 | 飞书热卖品类、FPS 场景、实拍首图 |
| Case 2 | 三模机械键盘 | 灯效素材、桌搭排版、上新文案 |
| Case 3 | 电竞椅 | 私聊转化、试坐疑虑、物流安装问题 |

## 5. 当前限制

1. 当前版本不真实请求飞书 API。
2. 素材为本地模拟视觉卡片，不是真实图片附件。
3. 文案为规则生成，语言多样性有限。
4. 排版为 HTML 预览，不导出 PNG。
5. 发布检查为基础规则，不等同于法律审核。

## 6. 下一步优化

- 接入真实飞书多维表格。
- 接入真实素材附件和图片搜索服务。
- 用 Canvas 导出朋友圈海报 PNG。
- 接入大模型做更自然的文案生成。
- 保存每次发布后的表现，用于长期学习。
