# METHOF.md

> 文件名按用户要求保留。此文件内容用于说明项目方法，等同于 `METHOD.md` 的面试过程版。

## 方法摘要

本项目把「电竞朋友圈种草」从简单文案生成升级成运营 Agent：

```text
飞书表格
-> 产品/素材/运营数据
-> 自动配图
-> 自动排版
-> 文案生成
-> 发布检查
-> 反馈学习
```

## 关键改动

1. 左侧链路加入接口契约，能说明每个节点的输入输出。
2. 增加飞书营销表连接配置，面试时能讲真实系统如何接表格。
3. 配图由“建议”改为“Agent 自动检索和选择素材”。
4. 增加排版预览，让输出更像可交付内容。
5. 增加运营指标调优，让 Agent 根据评论、私聊、成交学习反馈。

## 为什么这样设计

面试官更关注完整链路：数据从哪里来，素材怎么来，生成后怎么发布，发布后如何复盘。因此需要把 Agent 做成闭环，而不是只做 Prompt。

## 可替换为真实能力的位置

当前原型是本地模拟版，后续可替换：

- 飞书同步函数 -> 飞书开放平台 API
- 素材检索函数 -> 飞书附件/OSS/图片搜索 API
- 文案生成函数 -> LLM
- 排版函数 -> Canvas/Figma/图片合成 API
- 反馈学习函数 -> 数据分析服务或推荐策略模型
