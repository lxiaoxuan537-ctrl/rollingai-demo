# OPTIMIZATION.md

## 1. 目标

让 Agent 从运营反馈中学习，而不是每次重新生成。

## 2. 输入指标

| 指标 | 解释 |
|---|---|
| 曝光 | 朋友圈触达 |
| 点赞 | 内容吸引力 |
| 评论 | 话题性和疑问点 |
| 私聊 | 潜在购买意向 |
| 成交 | 转化结果 |
| 热卖品类 | 最近更容易卖的产品方向 |
| 潜客沟通摘要 | 用户真实疑虑 |

## 3. 计算指标

```text
互动率 = (点赞 + 评论 + 私聊) / 曝光
私聊率 = 私聊 / 曝光
成交率 = 成交 / 曝光
```

## 4. 学习规则

| 情况 | 调优建议 |
|---|---|
| 互动率低 | 文案开头增加具体玩家场景或提问 |
| 评论多于私聊 | 结尾引导“要不要我发实拍/链接” |
| 私聊多但成交少 | 补充价格、库存、试用方式、疑虑回答 |
| 潜客问实拍 | 首图优先用桌搭或细节实拍 |
| 潜客问活动 | 文案明确活动截止、赠品、库存 |
| 该品类近期热卖 | 更直接露出活动利益点 |

## 5. 当前原型实现

当前用 `learnFromOps()` 根据表单指标和模拟飞书数据生成建议。

真实上线后可以把每次发布结果写回飞书，形成长期样本库。
