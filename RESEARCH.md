# RESEARCH.md

## 1. 题目理解

面试题要求快速搭建一个 AI Agent 工具。第一版只做文案生成还不够完整，因为真实运营场景不只是“写一句话”，还包括素材、表格数据、发布表现和复盘调优。

因此本项目升级为：

> 电竞朋友圈种草运营 Agent：连接飞书营销表，读取产品和运营数据，自动找配图并排版，生成朋友圈文案，再根据评论、私聊、成交等指标学习反馈并给出优化建议。

## 2. 用户与场景

目标用户：

- 电竞品牌私域运营
- 电竞门店销售
- 社群运营
- 内容运营实习生

高频场景：

- 新品上架
- 活动促销
- 电竞外设种草
- 战队周边推荐
- 门店引导私聊或到店体验
- 复盘朋友圈内容效果

## 3. 升级后的关键痛点

1. 文案生成只是第一步，还需要真实素材支撑。
2. 运营素材分散在飞书表格、群文件、网盘或本地文件夹里。
3. 朋友圈发布后有点赞、评论、私聊、成交，但这些反馈没有反哺下一条内容。
4. 不同品类表现不同，例如鼠标可能评论多，电竞椅可能私聊多。
5. 潜在客户沟通里藏着下一条内容的重点，比如价格、库存、手感、尺寸、试用方式。

## 4. 解决思路

Agent 拆成 7 个节点：

1. 飞书营销表同步
2. 产品信息结构化
3. 运营反馈学习
4. 自动素材检索
5. 文案生成和去 AI 味
6. 自动排版
7. 发布检查和推荐

## 5. MVP 边界

当前原型不直接调用真实飞书接口，而是保留接口字段和模拟返回数据。这样可以在面试现场稳定演示完整逻辑。

真实上线时替换：

- 飞书同步节点 -> 飞书开放平台 API
- 素材检索节点 -> 飞书附件/OSS/图片搜索 API
- 文案生成节点 -> Dify/百炼/HiAgent/LLM
- 排版节点 -> Canvas/Figma/Canva/图片合成服务
